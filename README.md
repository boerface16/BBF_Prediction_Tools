# Fantasy Baseball H2H Points Prediction Tool

An ML pipeline that predicts ESPN Head-to-Head fantasy baseball points for the 2026 season. It processes Retrosheet play-by-play data (2015-2025) and Baseball Savant Statcast metrics, calculates 50+ advanced stats from raw CSVs, trains gradient-boosted models (XGBoost/LightGBM), and produces draft rankings with keeper evaluations.

## How to Run

The pipeline runs sequentially — each phase depends on the previous one. You can run it in two ways:

### Option A: Run from Python (recommended for full pipeline)

```python
from src.data_builder import load_config, build_batters, build_pitchers
from src.advanced_stats import build_advanced_batters, build_advanced_pitchers
from src.re24_engine import build_re24_outputs
from src.espn_points_mapper import build_espn_points
from src.feature_builder import (build_batter_features, build_pitcher_features, generate_scatter_plots,
                                generate_regression_scatter_plots, generate_pair_plots, generate_re24_scatter)
from src.model_trainer import train_pipeline
from src.predictor import build_predictions
from src.keeper_evaluator import build_keeper_rankings

cfg = load_config()

# Phase 1: Aggregate game data to season level
build_batters(cfg)
build_pitchers(cfg)

# Phase 2: Calculate advanced stats (rates, park factors, OPS+, ERA-, FIP-)
build_advanced_batters(cfg)
build_advanced_pitchers(cfg)

# Phase 3: RE24 engine (reads 9 GB plays.csv — takes several minutes)
build_re24_outputs(cfg)

# Phase 4: ESPN fantasy points (the ML target variable)
build_espn_points(cfg)

# Phase 5: Feature engineering + Statcast merge
bat_feat = build_batter_features(cfg)
pit_feat = build_pitcher_features(cfg)
generate_scatter_plots(bat_feat, pit_feat, cfg)
generate_regression_scatter_plots(bat_feat, pit_feat, cfg)
generate_pair_plots(bat_feat, pit_feat, cfg)
generate_re24_scatter(cfg)

# Phase 6: Model training (Optuna tuning — takes several minutes)
train_pipeline("batter", cfg)
train_pipeline("pitcher", cfg)

# Phase 7: Draft rankings
build_predictions(cfg)

# Phase 8: Keeper evaluations
build_keeper_rankings(cfg)
```

## Pipeline Phases

### Phase 1 — Data Build
Aggregates game-level Retrosheet batting/pitching to season-level stats. Derives W/L/SV from boolean flags in the pitching data. Merges player names and primary positions from `allplayers.csv`. Flags players below qualification thresholds (200 PA batters, 45 IP pitchers; reduced for 2020 COVID season).

### Phase 2 — Advanced Stats
Calculates rate stats (AVG, OBP, SLG, OPS, ISO, BABIP, K%, BB%, ERA, WHIP, FIP, K/9, etc.), park factors from game-level run scoring (5-year rolling average), league averages per season, and park/league-adjusted metrics (OPS+, ERA-, FIP-).

### Phase 3 — RE24 Engine
Reads the 9 GB `plays.csv` in 1M-row chunks. Builds a 24-cell run expectancy matrix (8 base states x 3 outs) per season. Calculates RE24 per plate appearance, then aggregates per player-season. Derives wOBA linear weights from run values of each event type. Also calculates batted ball rates (GB%, FB%, LD%, HR/FB%).

### Phase 4 — ESPN Points Mapper
Applies ESPN H2H scoring to produce the ML target variable:
- **Batters:** `TB + R + RBI + BB + SB - SO`
- **Pitchers:** `IP*3 + K + W*5 + SV*5 + HD*2 - H - ER*2 - BB - L*2`

### Phase 5 — Feature Engineering
Merges Statcast metrics (barrel%, exit velo, xwOBA, sprint speed, xERA, weighted whiff%) via the ID crosswalk. Computes xFIP and xFIP- from batted ball data. Builds career rolling averages for regression signals (BABIP, HR/FB%). Constructs the year-N-features → year-N+1-target shifted dataset that prevents future data leakage. Generates 90+ scatter, regression, pair, and RE24 plots.

### Phase 6 — Model Training
Trains both XGBoost and LightGBM with multi-seed Optuna tuning (5 seeds × 100 trials each, median-Spearman seed selection) using expanding-window time-series cross-validation (3 folds). Blends tuned models into a simple-average ensemble. Batters train on 2015-2023, pitchers on 2017-2023, with separate SP/RP models. COVID 2020 samples are downweighted via sample weights. Reports top-N precision (top-25/50/100). Produces SHAP summary/beeswarm plots and Pareto feature importance charts. Saves both trained models and ensemble metadata to `outputs/`.

### Phase 7 — Predictions & Draft Rankings
Generates 2026 projections from 2025 features. Calculates PAR (Points Above Replacement) by position, assigns tiers based on PAR gaps, and adds risk flags (BABIP regression, ERA regression, age). Saves ranked CSVs and 4 evaluation plots (predicted vs actual, residuals, position accuracy, tier distribution).

### Phase 8 — Keeper Evaluator
Loads keeper candidates from `data/keepers_candidates.txt`. Calculates a trajectory score from recent 2-year point history and a separate ML projection. Combines them 50/50 (configurable in `config.yaml`), flags signal divergence, and classifies each player as AUTO-KEEP / KEEP / BORDERLINE / CUT. Generates per-player trajectory plots.

## Configuration

All tunable parameters live in `config.yaml`:

- **Season ranges** — which years go into train/validation/test/predict splits (separate batter/pitcher windows)
- **Qualification thresholds** — minimum PA (batters) and IP outs (pitchers)
- **ESPN scoring weights** — point values for each stat category
- **Feature lists** — model input features under `features.batter.model` and `features.pitcher.model` (comment out with `#` to disable)
- **Keeper settings** — trajectory/ML weight split, divergence threshold, classification cutoffs
- **Model settings** — random seed, Optuna trial count, multi-seed tuning seeds, prediction bounds
- **COVID 2020 handling** — pro-rated target multiplier, sample weight for downweighting shortened-season data

## Known Limitations

- **Holds** are not tracked in Retrosheet data; set to 0. This undervalues middle relievers by roughly 0-40 points.
- **Pitcher model Spearman** (SP 0.505, RP 0.450) is below the 0.65 aspirational target. Pitcher year-over-year performance is inherently more volatile than batters due to injury risk and role changes. Multi-seed Optuna tuning reduces but doesn't eliminate stochasticity.
- **Batter model Spearman (0.613)** approaching the 0.65 target — improved via feature restoration, simple-average blending, and COVID downweighting.
- **COVID 2020** shortened season data is downweighted via sample weights; targets are pro-rated to 162-game equivalents.
- **Keeper candidates** with no Retrosheet data (e.g., prospects, name mismatches) are flagged as INSUFFICIENT_DATA.
